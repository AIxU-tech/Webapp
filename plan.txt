  # AIxU Frontend Restructuring Master Plan                                                                                              
                                                                                                                                         
  ## Overview                                                                                                                            
                                                                                                                                         
  This is the high-level master plan for restructuring the AIxU frontend codebase. It defines the target architecture, migration         
  strategy, and categorizes work into future detailed plans. **This document does NOT contain implementation details** - those will      
  be separate, focused plans.                                                                                                            
                                                                                                                                         
  ### Goals                                                                                                                              
  1. **Reduce redundancy** - Consolidate duplicate patterns (modals, buttons, forms)                                                     
  2. **Increase readability** - Organize 41 root components into logical directories                                                     
  3. **Modularize & reuse** - Extract shared utilities and enhance hook factories                                                        
  4. **Follow React best practices** - Feature-first organization, clean boundaries                                                      
                                                                                                                                         
  ### Current State Summary                                                                                                              
  - **176 files** across 22 directories                                                                                                  
  - **41 components at root level** with no organization (modals, cards, forms, layouts mixed)                                           
  - **4 different modal patterns** instead of one                                                                                        
  - **5 button components** used inconsistently + 15+ inline hardcoded buttons                                                           
  - **Duplicate logic** in file uploads, cache updates, form validation                                                                  
  - **Business logic leaking** into UI components (URL construction, tag constants)                                                      
                                                                                                                                         
  ### What's Working Well (Keep)                                                                                                         
  - Clean separation: API → Hooks → Components                                                                                           
  - React Query integration with optimistic updates                                                                                      
  - Query key factory pattern                                                                                                            
  - WebSocket integration for messaging                                                                                                  
  - Feature directories: `profile/`, `university/`, `messages/`                                                                          
  - Comprehensive barrel exports

  ## Categorized Future Detailed Plans                                                                                                   
                                                                                                                                         
  Each of these requires its own implementation plan with specific file changes:                                                         
                                                                                                                                         
  ### Category A: Directory Restructuring Plans                                                                                          
                                                                                                                                         
  **A1. UI Primitives Reorganization**                                                                                                   
  - Create subdirectories within `ui/` (buttons/, feedback/, modals/, forms/, display/, cards/, images/, lists/)                         
  - Move 30 existing ui/ components into appropriate subdirectories                                                                      
  - Update `ui/index.js` barrel export with sub-exports                                                                                  
                                                                                                                                         
  **A2. Layout Components Extraction**                                                                                                   
  - Create `components/layout/`                                                                                                          
  - Move: AppLayout, FeedPageLayout, AuthFormLayout, VerificationPageLayout, NavBar, Footer, PlasmaBackground                            
  - Update App.jsx and page imports                                                                                                      
                                                                                                                                         
  **A3. Auth Feature Directory**                                                                                                         
  - Create `components/auth/`                                                                                                            
  - Move: LoginModal, RegisterModal, AuthRoute, ProtectedRoute, EmailVerificationForm, NameInputPair, TermsModal, TermsModalWrapper,     
  TermsLink                                                                                                                              
                                                                                                                                         
  **A4. Community Feature Directory**                                                                                                    
  - Create `components/community/`                                                                                                       
  - Move: NoteCard, NotesFilter, CommentSection, CommentCard, SharePopover                                                               
  - Move CreateNoteModal FROM `ui/` TO here                                                                                              
                                                                                                                                         
  **A5. Opportunities & Events Directories**                                                                                             
  - Create `components/opportunities/` - move OpportunityCard, CreateOpportunityModal                                                    
  - Create `components/events/` - move EventCard, CreateEventModal                                                                       
                                                                                                                                         
  **A6. Admin, News & Shared Directories**                                                                                               
  - Create `components/admin/` - move RequestCard, CreateUniversityModal                                                                 
  - Create `components/news/` - move ContentCard                                                                                         
  - Create `components/shared/` - move UniversityCard, FeatureCard, AppPrefetcher                                                        
                                                                                                                                         
  ### Category B: Component Consolidation Plans                                                                                          
                                                                                                                                         
  **B1. Modal System Consolidation**                                                                                                     
  - Audit: BaseModal, ConfirmationModal, UnsavedChangesModal, custom implementations                                                     
  - Refactor all modals to extend BaseModal                                                                                              
  - Create standard modal API                                                                                                            
                                                                                                                                         
  **B2. Button Component Unification**                                                                                                   
  - Audit: GradientButton, SecondaryButton, IconButton, LikeButton, CloseButton + inline buttons                                         
  - Consider unified Button with variants                                                                                                
  - Replace 15+ inline hardcoded buttons with proper components                                                                          
                                                                                                                                         
  **B3. Form Field Components**                                                                                                          
  - Create FormField wrapper (label + input + error + help text)                                                                         
  - Integrate with useForm hook                                                                                                          
  - Standardize form validation patterns                                                                                                 
                                                                                                                                         
  **B4. State Display Consolidation**                                                                                                    
  - Create shared StateWrapper base                                                                                                      
  - Refactor EmptyState, LoadingState, ErrorState                                                                                        
                                                                                                                                         
  ### Category C: Hook & Utility Plans                                                                                                   
                                                                                                                                         
  **C1. useFileUpload Hook**                                                                                                             
  - Audit: Profile picture upload, banner upload, logo upload                                                                            
  - Create unified hook with progress, validation, cropping support                                                                      
                                                                                                                                         
  **C2. Hook Factory Expansion**                                                                                                         
  - Add createLikeHook to feedItemHooks factory                                                                                          
  - Reduce manual overrides in useNotes.js                                                                                               
                                                                                                                                         
  **C3. URL Construction Utilities**                                                                                                     
  - Extract URL building from UniversityHeroBanner, ProfileHeader                                                                        
  - Create `utils/url.js` with typed helpers                                                                                             
                                                                                                                                         
  **C4. Tag System Utilities**                                                                                                           
  - Consolidate LOCATION_TAGS, COMPENSATION_TAGS constants                                                                               
  - Create `utils/tags.js` with ordering and validation                                                                                  
                                                                                                                                         
  ### Category D: Code Quality Plans                                                                                                     
                                                                                                                                         
  **D1. Business Logic Extraction**                                                                                                      
  - Remove URL construction from UI components

  **D2. API Response Normalization**

  **D3. WebSocket Event Abstraction**
