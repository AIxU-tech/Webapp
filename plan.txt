  # AIxU Frontend Restructuring Master Plan                                                                                              
                                                                                                                                         
  ## Overview                                                                                                                            
                                                                                                                                         
  This is the high-level master plan for restructuring the AIxU frontend codebase. It defines the target architecture, migration         
  strategy, and categorizes work into future detailed plans. **This document does NOT contain implementation details** - those will      
  be separate, focused plans.                                                                                                            
                                                                                                                                         
  ### Goals                                                                                                                              
  1. **Reduce redundancy** - Consolidate duplicate patterns (modals, buttons, forms)                                                     
  2. **Increase readability** - Organize 41 root components into logical directories                                                     
  3. **Modularize & reuse** - Extract shared utilities and enhance hook factories                                                        
  4. **Follow React best practices** - Feature-first organization, clean boundaries                                                      
                                                                                                                                         
  ### Current State Summary                                                                                                              
  - **176 files** across 22 directories                                                                                                  
  - **41 components at root level** with no organization (modals, cards, forms, layouts mixed)                                           
  - **4 different modal patterns** instead of one                                                                                        
  - **5 button components** used inconsistently + 15+ inline hardcoded buttons                                                           
  - **Duplicate logic** in file uploads, cache updates, form validation                                                                  
  - **Business logic leaking** into UI components (URL construction, tag constants)                                                      
                                                                                                                                         
  ### What's Working Well (Keep)                                                                                                         
  - Clean separation: API → Hooks → Components                                                                                           
  - React Query integration with optimistic updates                                                                                      
  - Query key factory pattern                                                                                                            
  - WebSocket integration for messaging                                                                                                  
  - Feature directories: `profile/`, `university/`, `messages/`                                                                          
  - Comprehensive barrel exports

  ## Guiding Principles                                                                                                                  
                                                                                                                                         
  ### 1. Feature-First Organization                                                                                                      
  Components organized by feature, not by type. If a component is used exclusively by one feature, it belongs in that feature's          
  directory.                                                                                                                             
                                                                                                                                         
  ### 2. UI Primitives = No Business Logic                                                                                               
  The `ui/` directory contains ONLY stateless, generic primitives. If it imports from `api/`, `hooks/`, or `contexts/`, it's NOT a       
  UI primitive.                                                                                                                          
                                                                                                                                         
  ### 3. Barrel Exports at Feature Boundaries                                                                                            
  Each feature directory has an `index.js` that exports its public API. Internal components are NOT exported.                            
                                                                                                                                         
  ### 4. Co-locate Feature-Specific Constants                                                                                            
  Tag constants, helpers specific to a feature stay with that feature (e.g., `opportunities/constants.js`).                              
                                                                                                                                         
  ### 5. Extract Shared Utilities Proactively                                                                                            
  When logic is used by 2+ features, extract to `utils/`.                                                                                
                                                                                                                                         
  ---                                                                                                                                    
                                                                                                                                         
  ## Migration Strategy: Incremental                                                                                                     
                                                                                                                                         
  **Approach:** Create new structure alongside existing, move components one feature at a time, update imports immediately after         
  each move. No "big bang" cutover.                                                                                                      
                                                                                                                                         
  ### Phase Sequence                                                                                                                     
                                                                                                                                         
  | Phase | Scope | Description |                                                                                                        
  |-------|-------|-------------|                                                                                                        
  | **0** | Infrastructure | Add Vite path aliases, create directory skeleton |                                                          
  | **1** | UI Primitives | Organize `ui/` into subdirectories (buttons/, feedback/, etc.) |                                             
  | **2** | Layout | Create `layout/`, move AppLayout, NavBar, Footer, etc. |                                                            
  | **3** | Auth Feature | Create `auth/`, move LoginModal, RegisterModal, AuthRoute, etc. |                                             
  | **4** | Community Feature | Create `community/`, move NoteCard, CreateNoteModal, etc. |                                              
  | **5** | Opportunities & Events | Create both, move OpportunityCard, EventCard, etc. |                                                
  | **6** | Remaining Features | Create `admin/`, `news/`, `shared/` |                                                                   
  | **7** | Hook & Utils | Add useFileUpload, url.js, tags.js |                                                                          
  | **8** | Cleanup | Remove empty dirs, update docs |
